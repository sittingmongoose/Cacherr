## Documentation Standards

All documentation follows these standards:
- Written in Markdown format
- Includes code examples where applicable
- Contains table of contents for longer documents
- Uses consistent formatting and structure
- Regularly updated with code changes

## Contributing to Documentation

When contributing to the codebase, please ensure documentation is updated accordingly:

1. Update relevant existing documentation
2. Add new documentation for new features
3. Update API documentation for endpoint changes
4. Create ADRs for significant architectural decisions

For detailed contribution guidelines, see [Contributing Guide](development/contributing.md).
